Подробное описание

Функции трансивера, которые есть смысл вызывать только в режиме передатчика.

◆ nrf24l01_tx_configure()

int nrf24l01_tx_configure ( struct nrf24l01_tx_config *  config)

Инициализирует трансивер как передатчик.

Аргументы
configАдрес структуры с конфигурацией передатчика.
Возвращает
NRF24L01_OK, если инициализация прошла успешно,
NRF24L01_EBUS, в противном случае.

◆ nrf24l01_tx_configure_default()

int nrf24l01_tx_configure_default ( void  )

Инициализирует трансивер как передатчик настройками по-умолчанию.

Минимальная конфигурация передатчика и приёмника позволяет протестировать их работоспособность, используя самые базовые их возможности: односторонний обмен данными без подтверждений и повторных отправок.

Внимание
Данная функция будет корректно работать только если настройки трансивера ранее не менялись (со времени подачи на него питания).
Возвращает
NRF24L01_OK, если инициализация прошла успешно,
NRF24L01_EBUS, в противном случае.

◆ nrf24l01_tx_reuse_pld()

void nrf24l01_tx_reuse_pld ( void  )

Заставляет передатчик не удалять пакет из очереди отправки и при следующем вызове nrf24l01_tx_transmit() отправлять его же.

Заметки
Вернуть стандартное поведение при отправке можно вызовом nrf24l01_tx_write_pld() или nrf24l01_flush_tx_fifo().

◆ nrf24l01_tx_start_cont_transmission()

void nrf24l01_tx_start_cont_transmission ( void  )

Включает у передатчика режим постоянной отправки.

В этом режиме передатчик отправляет пакет, как только тот будет записан в очередь отправки, не дожидаясь вызова nrf24l01_tx_transmit(). Таким образом очередь отправки постоянно опустошается.

◆ nrf24l01_tx_transmit()

void nrf24l01_tx_transmit ( void  )

Инициирует отправку первого в очереди пакета.

Заметки
Функция удаляет отправленный пакет из очереди отправки, если до неё не было включено повторное использование пакетов вызовом nrf24l01_tx_reuse_pld().
Внимание
Если ранее был включён режим постоянной отправки, он будет отключён.
